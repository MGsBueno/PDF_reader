Language: English

Algorithm PDFBlockExtraction

Input:
    pdf_file_paths
    output_xml_path
    doc_type_config_path

Output:
    A single XML document containing the extracted logical blocks

Function LoadDocumentTypeConfig(doc_type_config_path)
    If doc_type_config_path does not exist
        Return an empty document type configuration

    Open the JSON file
    Read the configured blocks and ignore rules
    Return the document type configuration

Function ExtractLinesFromPdf(pdf_file_path)
    Open the PDF document
    For each page in the document
        Read page blocks as structured text
        For each text block
            If the block has no lines
                Continue
            For each line in the block
                Join all spans into a single line of text
                If the line is empty
                    Continue
                Collect:
                    text
                    maximum font size found in the line
                    whether any span uses a bold font
    Close the PDF document
    Return the ordered list of extracted lines

Function DetectBlock(line, document_type_config)
    If the line is not bold
        Return no block

    For each configured block rule
        For each regex pattern in the rule
            If the line text matches the pattern
               and the line font size is greater than or equal to the configured minimum
                Return the block name

    Return no block

Function ShouldIgnore(line_text, document_type_config)
    For each configured ignored prefix
        If line_text starts with the ignored prefix
            Return true
    Return false

Function ProcessPdfFiles(pdf_file_paths, output_xml_path, doc_type_config_path)
    document_type_config <- LoadDocumentTypeConfig(doc_type_config_path)
    Start XML document with root tag <dados>

    For each pdf_file_path in pdf_file_paths
        current_block_name <- null
        current_block_text <- empty string
        lines <- ExtractLinesFromPdf(pdf_file_path)

        For each line in lines
            If ShouldIgnore(line.text, document_type_config)
                If current_block_name is not null
                    Write current block to XML
                    current_block_name <- null
                    current_block_text <- empty string
                Continue

            detected_block_name <- DetectBlock(line, document_type_config)

            If detected_block_name exists
                If current_block_name is not null
                    Write current block to XML
                current_block_name <- detected_block_name
                current_block_text <- line.text
                Continue

            If current_block_name is not null
                Append line.text to current_block_text

        If current_block_name is not null
            Write current block to XML

    Finish XML document with closing tag </dados>

Cost Study

Definitions:
    P <- number of PDF files
    L <- total number of extracted lines across all PDF files
    B <- number of configured block types
    R <- average number of regex patterns per block
    I <- number of configured ignored prefixes
    S <- total number of parsed spans

Time Cost:
    Loading the JSON configuration:
        O(B * R + I)
    Extracting lines from PDFs:
        O(S)
    Ignoring checks for all lines:
        O(L * I)
    Block detection for all lines in the worst case:
        O(L * B * R)
    XML writing for all detected content:
        O(L)
    Total worst-case time cost:
        O(S + L * I + L * B * R)

Special Cases:
    If B and R are treated as constants:
        O(S + L)
    If the number of rules grows proportionally with the number of lines:
        O(L^2)

Memory Cost:
    Configuration in memory:
        O(B * R + I)
    Extracted lines kept in memory for one PDF:
        O(M)
    Current block accumulation:
        O(M) in the worst case
    Total memory cost:
        O(M + B * R + I)

Practical Notes:
    Increasing the number of block rules or regex patterns increases detection time
    Very large PDFs increase extraction time and temporary memory use
    Writing XML incrementally helps keep output memory cost low
    If needed, memory usage can be reduced further by streaming extracted lines instead of storing all lines first

Algoritmo ExtracaoDeBlocosPDF

Entrada:
    lista_de_arquivos_pdf
    caminho_do_xml_de_saida
    caminho_da_configuracao_doc_type

Saida:
    Um unico documento XML contendo os blocos logicos extraidos

Funcao CarregarConfiguracaoDeTipoDeDocumento(caminho_da_configuracao_doc_type)
    Se caminho_da_configuracao_doc_type nao existir
        Retornar uma configuracao vazia de tipo de documento

    Abrir o arquivo JSON
    Ler os blocos configurados e as regras de ignorar
    Retornar a configuracao de tipo de documento

Funcao ExtrairLinhasDoPdf(caminho_do_arquivo_pdf)
    Abrir o documento PDF
    Para cada pagina no documento
        Ler os blocos da pagina como texto estruturado
        Para cada bloco de texto
            Se o bloco nao tiver linhas
                Continuar
            Para cada linha no bloco
                Juntar todos os spans em uma unica linha de texto
                Se a linha estiver vazia
                    Continuar
                Coletar:
                    texto
                    maior tamanho de fonte encontrado na linha
                    se algum span usa fonte em negrito
    Fechar o documento PDF
    Retornar a lista ordenada de linhas extraidas

Funcao DetectarBloco(linha, configuracao_tipo_documento)
    Se a linha nao estiver em negrito
        Retornar nenhum bloco

    Para cada regra de bloco configurada
        Para cada expressao regular da regra
            Se o texto da linha corresponder ao padrao
               e o tamanho da fonte da linha for maior ou igual ao minimo configurado
                Retornar o nome do bloco

    Retornar nenhum bloco

Funcao DeveIgnorar(texto_da_linha, configuracao_tipo_documento)
    Para cada prefixo configurado para ignorar
        Se texto_da_linha comecar com o prefixo ignorado
            Retornar verdadeiro
    Retornar falso

Funcao ProcessarArquivosPdf(lista_de_arquivos_pdf, caminho_do_xml_de_saida, caminho_da_configuracao_doc_type)
    configuracao_tipo_documento <- CarregarConfiguracaoDeTipoDeDocumento(caminho_da_configuracao_doc_type)
    Iniciar documento XML com a tag raiz <dados>

    Para cada caminho_do_arquivo_pdf em lista_de_arquivos_pdf
        nome_bloco_atual <- nulo
        texto_bloco_atual <- string vazia
        linhas <- ExtrairLinhasDoPdf(caminho_do_arquivo_pdf)

        Para cada linha em linhas
            Se DeveIgnorar(linha.text, configuracao_tipo_documento)
                Se nome_bloco_atual nao for nulo
                    Escrever o bloco atual no XML
                    nome_bloco_atual <- nulo
                    texto_bloco_atual <- string vazia
                Continuar

            nome_bloco_detectado <- DetectarBloco(linha, configuracao_tipo_documento)

            Se nome_bloco_detectado existir
                Se nome_bloco_atual nao for nulo
                    Escrever o bloco atual no XML
                nome_bloco_atual <- nome_bloco_detectado
                texto_bloco_atual <- linha.text
                Continuar

            Se nome_bloco_atual nao for nulo
                Anexar linha.text ao texto_bloco_atual

        Se nome_bloco_atual nao for nulo
            Escrever o bloco atual no XML

    Finalizar documento XML com a tag de fechamento </dados>

Estudo de Custo

Definicoes:
    P <- numero de arquivos PDF
    L <- numero total de linhas extraidas em todos os arquivos PDF
    B <- numero de tipos de bloco configurados
    R <- numero medio de expressoes regulares por bloco
    I <- numero de prefixos configurados para ignorar
    S <- numero total de spans analisados

Custo de Tempo:
    Carregar a configuracao JSON:
        O(B * R + I)
    Extrair linhas dos PDFs:
        O(S)
    Verificar itens ignorados para todas as linhas:
        O(L * I)
    Detectar blocos para todas as linhas no pior caso:
        O(L * B * R)
    Escrever o XML para todo o conteudo detectado:
        O(L)
    Custo total de tempo no pior caso:
        O(S + L * I + L * B * R)

Casos Especiais:
    Se B e R forem tratados como constantes:
        O(S + L)
    Se o numero de regras crescer proporcionalmente ao numero de linhas:
        O(L^2)

Custo de Memoria:
    Configuracao em memoria:
        O(B * R + I)
    Linhas extraidas mantidas em memoria para um PDF:
        O(M)
    Acumulacao do bloco atual:
        O(M) no pior caso
    Custo total de memoria:
        O(M + B * R + I)

Observacoes Praticas:
    Aumentar o numero de regras de bloco ou expressoes regulares aumenta o tempo de deteccao
    PDFs muito grandes aumentam o tempo de extracao e o uso temporario de memoria
    Escrever o XML de forma incremental ajuda a manter baixo o custo de memoria da saida
    Se necessario, o uso de memoria pode ser reduzido ainda mais com extracao em fluxo, sem armazenar todas as linhas antes
